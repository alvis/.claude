      Semantics map · cursor port

      Prove the behavior before translating the syntax.

      This map isolates the cursor semantics in vendor/stream-cursor and tests how they survive in the TypeScript event worker. It does not claim the runtimes, storage engines, or concurrency models match.

      Input
      Partition, lease, delivery sequence
      Output
      Highest contiguous acknowledged sequence
      Failure
      Expired lease makes acknowledgment stale
      Target gap
      Worker stores only latest seen sequence

      The map claims only observable semantics supported by evidence.

      Source inventory

      Five observable promises, three incidental mechanisms.

      Evidence inspected

      cursor.go, lease.go, replay.go, and 18 behavior tests. The source uses PostgreSQL and goroutines; neither is part of the requested contract.

      1. Lease one partition

        One owner may advance a partition during a bounded lease epoch.

      2. Deliver beyond gaps

        Later messages may finish early, but the durable cursor waits for the gap.

      3. Commit contiguous progress

        Acknowledgment advances to the highest sequence with no missing predecessor.

      4. Reject stale owners

        An expired lease cannot mutate progress even if its work eventually completes.

      5. Replay explicitly

        Operators may move backward within retained history; forward skips are rejected.

      Semantic gotcha · number precision

      Go sequence values are uint64. A JavaScript number loses integer precision beyond 253−1, so the target must use bigint end to end. The loop shape is similar; the numeric contract is not.

      − cursor = greatest completed sequence
      + cursor = greatest contiguous completed sequence

      Semantic gotcha · two clocks, two jobs

      The in-process deadline uses a monotonic clock so wall-clock correction cannot extend ownership. Persistence still stores a wall-clock expiry for cross-process takeover. Collapsing both to Date.now() changes failure behavior during clock adjustment.

      Mapping 03

      Replay moves backward; it never invents a forward skip.

      Reference contract

      Bounded rewind

      The requested sequence must be no newer than committed and no older than retained history.

      Conformance ledger

      Preserve the contract; adapt or drop the machinery deliberately.

      Preserve exactly

      Contiguous commit

      Gaps block progress, epochs fence stale owners, and replay cannot jump forward.

      Adapt deliberately

      Error and clock surfaces

      Sentinel errors become typed errors; monotonic duration and persisted wall time remain separate.

      Drop as incidental

      Goroutine and SQL shape

      Channels, mutexes, and PostgreSQL syntax are implementation choices, not observable promises.

      One fence, four surfaces
