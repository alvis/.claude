What the plan said

Each breach sends one alert via sendAlert(fleetId, breach). No rate handling mentioned.

What the code revealed

sendAlert() fans out to SMS with no per-fleet ceiling. A vehicle parked on an edge with GPS jitter fires rapid enter/exit breaches — hundreds of texts an hour for one vehicleinvented, each billable.

The conservative choice

Added a per-fleet hourly alert budget; past the cap, breaches roll into one "N more suppressed" digest, mirroring the existing email digest cap in notify.
