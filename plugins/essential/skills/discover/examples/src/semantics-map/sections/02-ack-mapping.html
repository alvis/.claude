Semantic gotcha · number precision

Go sequence values are uint64. A JavaScript number loses integer precision beyond 253−1, so the target must use bigint end to end. The loop shape is similar; the numeric contract is not.

− cursor = greatest completed sequence
+ cursor = greatest contiguous completed sequence
