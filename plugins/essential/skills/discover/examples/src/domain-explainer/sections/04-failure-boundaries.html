Where it breaks

Three boundaries deserve explicit proof.

Open one boundary at a time — each names the failure, the proof it demands, and where that proof lives.

Mixed application versions

A rolling deploy means old and new binaries overlap. The schema must serve both until the final old instance is gone. observed

Proof: the compatibility adapter reads either shape. app/adapters/order_compat.rb:12-40

Partial backfill

Readers need a defined fallback when a record has not migrated, rather than assuming the new location is complete. assumed

This is the amber band in the rehearsal above: backfill gaps become quiet legacy reads only while the fallback stays on.

Silent divergence

Dual writes can disagree. Measure and reconcile the difference before changing which representation is authoritative. inferred

Proof: a mismatch counter compares both shapes on every write. app/models/order.rb:96-104

The irreversible moment comes last

Dropping the old column or table is the only step that removes an easy rollback. Schedule it after telemetry shows no supported code path reads or writes the legacy shape.

Walk the failing request path
API write → compatibility adapter → legacy column + new column → mismatch telemetry → rollback threshold. Expand this before implementation to verify every owner and fallback.
