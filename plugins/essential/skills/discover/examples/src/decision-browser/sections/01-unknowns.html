01 · Unknowns

What each work plan still hides

Every card names the unknown, the work it blocks, and a recommendation with its reasoning. Where real alternatives exist you choose between them; otherwise accept or override.

U1 · QuestDB version constraint Needs live query

Blocks — every ingest repair task. The crash loop is attributed to the 8.1.1 array codepath, but the fix strategy depends on which versions the host can actually run.

Evidence — the crash is reproduced in the ledger; the version constraint has never been queried against the live instance.

U2 · Do-not-edit fence Operator decision

Blocks — the scheduler build. Which files the automated coder may touch is undecided, so no task can be safely dispatched.

Recommendation — fence the strategy and risk-limit modules; everything else is editable. The strategy code encodes operator judgement no coder should rewrite.

Accept the recommended fence?

U3 · Lease duration Operator decision

Blocks — the scheduler's job-claim design. Too short and healthy jobs get pre-empted; too long and a dead worker parks its work for the whole lease.

Recommendation — 10-minute leases with heartbeat renewal. Matches the longest observed job runtime plus margin, and a dead worker frees its work within minutes.

Accept the recommended lease model?

U4 · LOB array encoding Real alternatives

Blocks — the ingest repair and every query-side consumer of order-book depth.

Evidence — three viable encodings with different read/write trade-offs; the choice shapes the schema the live probe must validate.

U5 · Paper-account credentials Operator decision

Blocks — the first paper trade. No task can place an order until the paper account exists and its credentials have a storage home.

Recommendation — provision the paper account now and store credentials in the same secrets path the ingest keys already use; no new mechanism.

Accept the recommended credential path?

U6 · Live ingest rate Needs live query

Blocks — capacity sizing for the repaired ingest. The design assumes the historical tick rate; the current rate has never been measured on the live feed.

Recommendation — sample the live feed for one session before sizing buffers; a read-only tap, no schema changes.

Accept the recommended sampling probe?
