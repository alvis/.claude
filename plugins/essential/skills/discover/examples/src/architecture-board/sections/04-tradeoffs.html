Self-critique

What this architecture buys, and where it breaks.

Trade-offs, honestly

Wins

  • A stateless gateway scales horizontally; any node can serve any session with no sticky routing.
  • An event-sourced op log makes every document state reconstructable and auditable by replay.
  • CRDT convergence removes the central merge lock, so a slow client never blocks everyone else's edits.

Costs

  • CRDT metadata grows per edit, so memory and payload cost more than compact OT deltas.
  • Snapshot compaction adds a background job and a window where a snapshot lags the authoritative log.

Fails when

  • A single hot document's in-memory CRDT state outgrows one service instance's memory budget.
  • Presence fan-out over one Redis channel saturates at very high concurrent-editor counts.

Data notes

The latency-budget proportions, the few-thousand live-ops figure, and any throughput or concurrency numbers implied by the diagram are invented illustrative shapes. Treat them as placeholders, not measured Conflux numbers, when building the service.
