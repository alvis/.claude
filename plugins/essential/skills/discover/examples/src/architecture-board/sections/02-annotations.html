What each pin defends

Four decisions worth pinning to the diagram.

  1. The gateway is stateless. The Realtime Gateway holds no document state, so any node can serve any session and the edge scales horizontally behind a plain load balancer decided.
  2. CRDT merge over OT. The Merge Engine uses a Yjs-style CRDT so concurrent edits converge without a central merge lock — trading extra per-edit metadata for a simpler, lock-free server approved.
  3. The op log is the source of truth. Every state is reconstructable by replaying the append-only Op Log; the Snapshot Store is a derived cache, never authoritative decided.
  4. Presence is ephemeral. Cursors and selections fan out over Redis pub/sub and are never persisted; whether one channel holds up at peak concurrency is unproven open question.
