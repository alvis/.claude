What this plan deliberately leaves out.

These are my non-goals and soft spots for this pass — not the cost of any one option, which lives in each step's trade-offs. Read them before you treat the plan as complete.

Left out on purpose

  • No automatic duplicate detection. A human still decides two threads are the same; suggesting merges from text similarity is a separate, bigger project.
  • No bulk merge. This ships one-at-a-time merges only; clearing a backlog of duplicates in one action is noted as a follow-up, not built.
  • No cross-product merges. Threads can only merge within the same product — merging across products raises permission questions I did not want to answer implicitly.

Weakest part of this plan

  • Merge chains. If A is merged into B and later B into C, the redirect-tombstone option walks the chain correctly but stores no summary of it — deep chains are rare but the plan does not prove they stay cheap to resolve.
  • The "9 customers voted on both" figure in step 2 is an illustrative shape for showing the dedupe math, not a measured overlap rate — it must not be used to size anything. overlap rate
