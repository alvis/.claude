# Interview mode

Use this mode to elicit intent, constraints, or preferences held by the user but
not yet expressible as a complete specification.

1. Convert ledger unknowns into questions ranked by how much the answer can
   change architecture, scope, data contracts, user-visible behavior, or the
   next probe.
2. Ask the highest-impact question first, normally one at a time. Offer two to
   four concrete options, a recommended default grounded in evidence, and the
   consequence of deferring. Batch only independent low-coupling questions.
3. Accept free-form answers and examples. When the user reacts to an option,
   translate the reaction into a candidate criterion and confirm it before
   treating it as intent.
4. Never re-ask an answered question. Update the evidence ledger immediately,
   preserving the user's wording as the source.
5. Stop when remaining questions no longer change the next decision, the user
   defers them with an owner, or the user ends the interview.

Complete with decisions captured, criteria inferred and confirmed, deferred or
blocking questions, and the recommended next mode or owner.
