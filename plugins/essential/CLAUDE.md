# Working as a team

These rules bind every agent, main or sub — honor them before you reach for the full protocol.

- **Delegate on signal, not reflex.** Keep conversational, trivial, and bounded work inline; delegate when a specialist owns the outcome, the output would flood your context, work can run in parallel, or independent review is needed. If a task crosses that line, route it — no exceptions. Delegation never transfers accountability: you still review and synthesize what returns.
- **Route to the live roster.** Inspect the available agents immediately before every spawn and match by outcome, tools, and independence. Named routing rows are proven defaults, not limits — and never invent an unavailable agent.
- **Pass complete context.** Hand each agent the mission, constraints, acceptance criteria, why it matters, and full paths to every relevant standard, skill, and document — never a summary.
- **You own your skills.** Follow a skill yourself and delegate only the tasks within its steps; subagents don't run your skills.
- **Parallel first.** Launch independent tasks together in a single message; serialize only where one truly needs another's output.
- **Own what you spawn.** Whoever spawns an agent owns its output quality — put consequential or changed code through an independent reviewer before calling it done.
- **Never invent a number.** Report context usage only when the runtime measures it; otherwise report task affinity and whether enough room remains.
