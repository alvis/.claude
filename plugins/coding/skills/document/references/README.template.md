# README template

This compatibility entrypoint indexes the complete bundled README template. Load every child in order when the bundled template is selected:

- [Authoring contract](readme-template/10-authoring-contract.md)
- [Introduction, quick start, and positioning](readme-template/20-introduction-and-positioning.md)
- [Concepts, configuration, relationships, architecture, and support](readme-template/30-concepts-architecture-and-support.md)
- [Usage and API reference](readme-template/40-usage-and-api.md)
- [Advanced usage, alternatives, ecosystem, FAQ, and operations](readme-template/50-advanced-and-operations.md)

The children are one template, not optional standalone variants. Omit unsupported sections rather than inventing content. Generated package documentation uses the repository's existing compatible path; for a new artifact, use lowercase `readme.md`.
