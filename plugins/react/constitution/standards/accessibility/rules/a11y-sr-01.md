# A11Y-SR-01: Alt Text for Images

## Intent

Informative images need descriptive `alt` text so screen reader users get the same information sighted users do. Decorative images need `alt=""` so they're skipped, not announced as "image".

## Fix

- For informative images, write `alt` text describing the *information* the image conveys (not "image of…")
- For decorative images, use `alt=""` (empty, not missing)
- For images-as-links/buttons, the `alt` text must describe the destination/action

```typescript
// ❌ BAD: missing alt text
<img src="chart.png" />

// ✅ GOOD: descriptive alt text
<img src="chart.png" alt="Sales increased 20% from Q1 to Q2" />
```

## Code Superpowers

- Grep for `<img` without an `alt` attribute
- Audit `alt=""` usage to confirm those images really are decorative
- Check chart/data visualizations for either a textual `alt` summary or an adjacent data table

## Common Mistakes

1. `alt="image"` or `alt="photo"` — adds noise without information
2. Missing `alt` entirely (screen readers fall back to filename)
3. Decorative icons inside `<button>` without `aria-hidden="true"` (announced twice)
